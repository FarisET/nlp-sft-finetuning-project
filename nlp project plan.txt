# LLM Fine-Tuning Assignment — Implementation Plan

## Context
This assignment implements a full SFT → DPO fine-tuning pipeline on TinyLlama-1.1B. The project is run on Google Colab in phases (baseline → SFT → DPO), with a parallel report maintained in Markdown. The directory `E:\Dev\NLP Finetuning project` is empty — everything is created from scratch.

---

## Chosen Stack
| Component | Choice | Reason |
|---|---|---|
| Base model | `TinyLlama/TinyLlama-1.1B-intermediate-step-1431k-3T` | Fast on Colab free tier, Llama architecture, wide LoRA support |
| SFT dataset | `tatsu-lab/alpaca` (52k samples, subset of 10k) | Classic beginner-friendly instruction dataset |
| DPO dataset | `Intel/orca_dpo_pairs` (~12.8k, subset of 3k) | Clean chosen/rejected format, works directly with TRL DPOTrainer |
| Training lib | `trl` (SFTTrainer + DPOTrainer) | Industry standard, minimal boilerplate |
| PEFT | `peft` LoRA | Low memory footprint for Colab |
| Quantization | `bitsandbytes` 4-bit NF4 | Fits 1.1B model on Colab T4 |
| Evaluation | `evaluate` (BLEU) + `bert-score` | Required by assignment |

## Phase 1 — Notebook 00: Baseline Evaluation

**File:** `00_baseline_evaluation.ipynb`

### Cells outline:
1. **Install** — `transformers bitsandbytes accelerate evaluate bert-score sacrebleu`
2. **Load model** — TinyLlama-1.1B in 4-bit NF4, `AutoTokenizer`
3. **Test prompts** — 10 prompts hardcoded (see table below), loaded from `test_prompts.json`
4. **Generation** — `model.generate()` with `max_new_tokens=200`, `temperature=0.7`, `do_sample=True`
5. **BLEU eval** — `evaluate.load("sacrebleu")`, compute sentence-level BLEU per prompt
6. **BERTScore eval** — `bert_score.score()` with `lang="en"`, rescale_with_baseline=True
7. **Results table** — Print and save DataFrame with prompt, generated, BLEU, BERTScore-F1
8. **Save** — Export `baseline_results.csv` to Google Drive

## Phase 2 — Notebook 01: SFT Training

**File:** `01_sft_training.ipynb`

### Cells outline:
1. **Install** — `transformers peft trl bitsandbytes accelerate datasets evaluate bert-score`
2. **Load dataset** — `tatsu-lab/alpaca`, shuffle, take first **10,000 samples**, split 90/10 train/val
3. **Format function** — convert alpaca `{instruction, input, output}` to single prompt string:
   ```
   Below is an instruction that describes a task. Write a response that appropriately completes the request.

   ### Instruction:
   {instruction}

   ### Input:
   {input}  (omit if empty)

   ### Response:
   {output}
   ```
4. **Base model + tokenizer** — 4-bit NF4 BnB config, `max_seq_length=512`
5. **Trial loop** — iterate over `TRIAL_CONFIGS` list (see below), for each:
   - Build `LoraConfig`
   - Build `TrainingArguments`
   - Instantiate `SFTTrainer`
   - Train and capture `val_loss`
   - Run eval on 10 test prompts → BLEU + BERTScore
   - Save adapter to `/content/drive/MyDrive/sft_trial_{n}/`
   - Append row to results DataFrame
6. **Best model selection** — rank by BERTScore-F1 (primary), BLEU (secondary), val_loss (tiebreaker)
7. **Results table** — print full comparison, state winner

### 5 SFT Trial Configurations:
| Trial | r | lora_alpha | target_modules | lr | batch_size | grad_accum | epochs | Notes |
|-------|---|------------|----------------|-----|------------|------------|--------|-------|
| T1 | 4 | 8 | `q_proj, v_proj` | 2e-4 | 4 | 4 | 1 | Minimal LoRA, baseline config |
| T2 | 8 | 16 | `q_proj, v_proj` | 2e-4 | 4 | 4 | 2 | Same modules, more epochs |
| T3 | 16 | 32 | `q_proj, k_proj, v_proj, o_proj` | 1e-4 | 8 | 2 | 2 | More attention modules |
| T4 | 32 | 64 | `q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj` | 5e-5 | 4 | 4 | 2 | Full LoRA, low LR |
| T5 | 8 | 16 | `q_proj, v_proj` | 5e-4 | 8 | 2 | 1 | High LR experiment |

All trials: `lora_dropout=0.05`, `bias="none"`, `task_type="CAUSAL_LM"`, `fp16=True`, `warmup_ratio=0.03`, `lr_scheduler_type="cosine"`, `weight_decay=0.01`.

## Phase 3 — Notebook 02: DPO Training

**File:** `02_dpo_training.ipynb`

### Cells outline:
1. **Install** — same as SFT notebook
2. **Load best SFT model** — load adapter from Drive path saved in Phase 2
3. **Load DPO dataset** — `Intel/orca_dpo_pairs`, take **3,000 samples**, split 90/10 train/val
4. **Format DPO data** — extract `prompt`, `chosen`, `rejected` columns into DPOTrainer-compatible format
5. **Reference model** — load frozen copy of SFT model as `ref_model` (required by DPO)
6. **Trial loop** — 5 DPO configs (see below):
   - Build `LoraConfig` (r=8, same target modules as best SFT trial)
   - Build `DPOConfig` with beta + training args
   - Instantiate `DPOTrainer`
   - Train, capture val_loss
   - Eval on 10 test prompts → BLEU + BERTScore
   - Save to Drive
7. **Best DPO selection** — same ranking as SFT
8. **Three-way comparison table** — Base vs Best SFT vs Best DPO on all 10 prompts

### 5 DPO Trial Configurations:
| Trial | beta | lr | batch_size | grad_accum | epochs | Notes |
|-------|------|-----|------------|------------|--------|-------|
| D1 | 0.1 | 1e-5 | 2 | 4 | 1 | Conservative start |
| D2 | 0.1 | 5e-5 | 4 | 2 | 2 | More epochs |
| D3 | 0.3 | 1e-5 | 2 | 4 | 1 | Higher beta (more conservative) |
| D4 | 0.5 | 2e-5 | 4 | 2 | 2 | High beta + more epochs |
| D5 | 0.1 | 1e-4 | 2 | 4 | 1 | High LR with low beta |

All DPO trials inherit the LoRA rank/modules from the best SFT trial.
